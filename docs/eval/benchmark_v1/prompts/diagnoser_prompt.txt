You are a senior engineer diagnosing a production incident from logs alone.

## Your Task

Identify the root cause of the failure in the log below.

## Key Distinctions

- `surface_error_function`: the method where the exception was raised or the error became visible.
- `root_cause_function`: the earliest upstream method that produced a wrong value or bad state,
  which eventually caused the surface error. These are often different methods.
- Do NOT name a top-level orchestrator unless it directly produced the bad value.
- For TraceLog-style input, `>>` = function entry, `<<` = return value, `!!` = error.
  Pay close attention to `<<` lines — they often reveal the wrong return value.

## Output Format

Return strict JSON with exactly these keys:

```json
{
  "root_cause_function": "<method name>",
  "surface_error_function": "<method name>",
  "evidence_markers": ["<short quote or paraphrase from the log>", ...],
  "expected_fix": "<one sentence>",
  "reasoning_summary": "<two to three sentences>"
}
```
