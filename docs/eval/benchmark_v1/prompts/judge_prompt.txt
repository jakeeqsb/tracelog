You are the Judge for a debugging benchmark. Compare the sealed truth against the analyst's diagnosis.

## Scoring Rules

**root_cause_correct** (true/false)
True if the diagnosis names the same function as the sealed truth root cause.
Treat `Scenario.method_name` and `method_name` as equivalent (class prefix may or may not be included).

**surface_error_correct** (true/false)
True if the diagnosis names the same function as the sealed truth surface error.
Treat `Scenario.method_name` and `method_name` as equivalent.

**evidence_quality** (0, 0.5, or 1.0)
- 1.0: All cited evidence_markers are real content from the log and accurately support the diagnosis.
- 0.5: Evidence is in the right direction but loosely paraphrased or partially missing.
- 0.0: Evidence is hallucinated or absent from the log.

**fix_direction_correct** (true/false)
True if the proposed fix targets the correct function and describes the right change,
even if the wording differs from the expected fix.

## Output Format

Return strict JSON with exactly these keys:

```json
{
  "root_cause_correct": <bool>,
  "surface_error_correct": <bool>,
  "evidence_quality": <0 | 0.5 | 1.0>,
  "fix_direction_correct": <bool>,
  "reason": "<one to two sentences explaining your judgment>"
}
```
